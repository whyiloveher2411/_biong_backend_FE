[h1] Strix
[h3] The open-source AI pentesting tool. Autonomous AI hackers that find and fix your app’s vulnerabilities.
[p] Tip
[strong] New!
[p] Strix integrates seamlessly with GitHub Actions and CI/CD pipelines. Automatically scan for vulnerabilities on every pull request and block insecure code before it reaches production -
[a] Get started with no setup required
[h2] Strix Overview
[p] Strix are autonomous AI penetration testing agents that act just like real hackers - they run your code dynamically, find vulnerabilities, and validate them through actual proofs-of-concept. Built for developers and security teams who need fast, accurate security testing without the overhead of manual pentesting or the false positives of static analysis tools.
[strong] Key Capabilities:
[strong] Full pentesting toolkit
[li] - reconnaissance, exploitation, and validation out of the box
[strong] Multi-agent orchestration
[li] - teams of AI pentesters that collaborate and scale
[strong] Real exploit validation
[li] - working PoCs, not false positives like legacy vulnerability scanners
[strong] Developer‑first CLI
[li] - actionable findings with remediation guidance
[strong] Auto‑fix & reporting
[li] - generate patches and compliance-ready pentest reports
[h2] Use Cases
[strong] Application Security Testing
[li] - Detect and validate critical vulnerabilities in your applications
[strong] Rapid Penetration Testing
[li] - Get penetration tests done in hours, not weeks, with compliance reports
[strong] Bug Bounty Automation
[li] - Automate bug bounty research and generate PoCs for faster reporting
[strong] CI/CD Integration
[li] - Run tests in CI/CD to block vulnerabilities before reaching production
[h2] 🚀 Quick Start
[strong] Prerequisites:
[li] Docker (running)
[li] An LLM API key from any
[a] supported provider
[li] (OpenAI, Anthropic, Google, etc.)
[h3] Installation & First Scan
[span] Install Strix
[pre] curl -sSL https://strix.ai/install
[pre] bash
[span] Configure your AI provider
[span] export
[pre] STRIX_LLM=
[span] openai/gpt-5.4
[span] export
[pre] LLM_API_KEY=
[span] your-api-key
[span] Run your first security assessment
[pre] strix --target ./app-directory
[p] Note
[p] First run automatically pulls the sandbox Docker image. Results are saved to
[code] strix_runs/<run-name>

[h2] ✨ Features
[h3] Agentic Pentesting Tools
[p] Strix agents come equipped with a comprehensive offensive security toolkit - the same tools used by professional penetration testers and ethical hackers:
[strong] HTTP Interception Proxy
[li] - Full request/response manipulation and analysis with Caido
[strong] Browser Exploitation
[li] - Automated browser for testing XSS, CSRF, clickjacking, and auth bypass flows
[strong] Shell & Command Execution
[li] - Interactive terminal for exploit development and post-exploitation
[strong] Custom Exploit Runtime
[li] - Python sandbox for writing and validating proof-of-concept exploits
[strong] Reconnaissance & OSINT
[li] - Automated attack surface mapping, subdomain enumeration, and fingerprinting
[strong] Static & Dynamic Code Analysis
[li] - SAST + DAST capabilities for comprehensive application security testing
[strong] Vulnerability Knowledge Base
[li] - Structured findings with CVSS scoring and OWASP classification
[h3] Comprehensive Vulnerability Scanner
[p] Strix identifies, validates, and exploits a wide range of security vulnerabilities across the OWASP Top 10 and beyond:
[strong] Broken Access Control
[li] - IDOR, privilege escalation, auth bypass
[strong] Injection Attacks
[li] - SQL injection, NoSQL injection, OS command injection, SSTI
[strong] Server-Side Vulnerabilities
[li] - SSRF, XXE, insecure deserialization, RCE
[strong] Client-Side Attacks
[li] - XSS (stored/reflected/DOM), prototype pollution, CSRF
[strong] Business Logic Flaws
[li] - Race conditions, payment manipulation, workflow bypass
[strong] Authentication & Session
[li] - JWT attacks, session fixation, credential stuffing vectors
[strong] Infrastructure & Cloud
[li] - Misconfigurations, exposed services, cloud security issues
[strong] API Security
[li] - Broken authentication, mass assignment, rate limiting bypass
[h3] Graph of Agents (Multi-Agent Pentesting)
[p] Advanced multi-agent orchestration for comprehensive automated penetration testing:
[strong] Distributed Pentesting
[li] - Specialized AI agents for recon, exploitation, and post-exploitation
[strong] Scalable Security Testing
[li] - Parallel execution across multiple targets for fast, comprehensive coverage
[strong] Dynamic Coordination
[li] - Agents share discoveries, chain vulnerabilities, and collaborate like a red team
[h2] Usage Examples
[h3] Basic Usage
[span] Scan a local codebase
[pre] strix --target ./app-directory
[span] Security review of a GitHub repository
[pre] strix --target https://github.com/org/repo
[span] Black-box web application assessment
[pre] strix --target https://your-app.com
[h3] Advanced Testing Scenarios
[span] Grey-box authenticated testing
[pre] strix --target https://your-app.com --instruction
[span] Perform authenticated testing using credentials: user:pass
[span] Multi-target testing (source code + deployed app)
[pre] strix -t https://github.com/org/app -t https://your-app.com
[span] Targets from a file, one target per non-empty, non-comment line
[pre] strix --target-list ./targets.txt
[span] White-box source-aware scan (local repository)
[pre] strix --target ./app-directory --scan-mode standard
[span] Focused testing with custom instructions
[pre] strix --target api.your-app.com --instruction
[span] Focus on business logic flaws and IDOR vulnerabilities
[span] Provide detailed instructions through file (e.g., rules of engagement, scope, exclusions)
[pre] strix --target api.your-app.com --instruction-file ./instruction.md
[span] Force PR diff-scope against a specific base branch
[pre] strix -n --target ./ --scan-mode quick --scope-mode diff --diff-base origin/main
[h3] Headless Mode
[p] Run Strix programmatically without interactive UI using the
[code] -n/--non-interactive
[p] flag - perfect for servers and automated jobs. The CLI prints real-time vulnerability findings and the final report before exiting. Exits with non-zero code when vulnerabilities are found.
[pre] strix -n --target https://your-app.com
[h3] CI/CD (GitHub Actions)
[p] Strix can be added to your pipeline to run a security test on pull requests with a lightweight GitHub Actions workflow:
[span] name
[span] strix-penetration-test
[span] pull_request
[span] jobs
[span] security-scan
[span] runs-on
[span] ubuntu-latest
[span] steps
[pre] :
      -
[span] uses
[span] actions/checkout@v6
[span] with
[span] fetch-depth
[span] name
[span] Install Strix
[span] run
[span] curl -sSL https://strix.ai/install | bash
[span] name
[span] Run Strix
[span] env
[span] STRIX_LLM
[span] ${{ secrets.STRIX_LLM }}
[span] LLM_API_KEY
[span] ${{ secrets.LLM_API_KEY }}
[span] run
[span] strix -n -t ./ --scan-mode quick
[p] Tip
[p] In CI pull request runs, Strix automatically scopes quick reviews to changed files.
If diff-scope cannot resolve, ensure checkout uses full history (
[code] fetch-depth: 0
[p] ) or pass
[code] --diff-base
[p] explicitly.
[h3] Configuration
[span] export
[pre] STRIX_LLM=
[span] openai/gpt-5.4
[span] export
[pre] LLM_API_KEY=
[span] your-api-key
[span] Optional
[span] export
[pre] LLM_API_BASE=
[span] your-api-base-url
[span] if using a local model, e.g. Ollama, LMStudio
[span] export
[pre] PERPLEXITY_API_KEY=
[span] your-api-key
[span] for search capabilities
[span] export
[pre] STRIX_REASONING_EFFORT=
[span] high
[span] control thinking effort (default: high, quick scan: medium)

[p] Strix automatically saves your configuration to
[code] ~/.strix/cli-config.json
[p] , so you don't have to re-enter it on every run.
